# Architecture

[English](architecture.md) | [日本語](architecture.ja.md)

## World state

`SilenceWorld` holds employees keyed by a sorted `BTreeMap<AgentId, Employee>` (sorted keys guarantee a deterministic activation order), the team table, the inter-employee `SocialNetwork` (Watts–Strogatz by default), the issue salience `σ(t)`, the whole-organisation climate of silence `C(t)`, and the per-step retaliation buffer.

Each `Employee` carries the persistent IVT cognitive structure — `ivt_strength` ι_i ∈ [0,1] and a 5-vector `ivt_rule_weights` (a simplex over the five rules `target_id`, `need_data`, `no_bypass`, `no_embarrass`, `career_consq`) — plus the contextual drivers (`fear`, `psych_safety`, `perceived_silence`, `voice_threshold`, `private_concern`), the current `expression` (VOICE / SILENCE / NEUTRAL), the silence `motive`, and `last_active_rules` (the IVT rule ids that fired in the most recent decision).

## 9 mechanisms × 6 phases

The socsim fixed loop is `PreStep → Environment → Decision → Interaction → Reward → PostStep`. Nine mechanisms are placed across it:

| # | Mechanism | Phase | Role |
|---|-----------|-------|------|
| 1 | `IssueSalience` | Environment | Mean-revert σ(t); fire an exogenous shock at `shock_t`. |
| 2 | `RetaliationEvent` | Environment | With probability `p_retaliate` mark agents touched by a retaliation event. |
| 3 | `FearAppraisal` | Decision | Threat appraisal: `f_i ← clamp(f_i + α·retaliated − γ·max(u,0))`. |
| 4 | `VoiceDecisionRule` / `VoiceDecisionLlm` | Decision | **Mutually exclusive.** Logit (rule / rule_no_ivt) vs LLM 5-rule self-reflection. |
| 5 | `SilenceSpiral` | Interaction | ρ_i ← neighbour silence ratio (Noelle-Neumann 1974). |
| 6 | `PrefalseCascade` | Interaction | A *falsely* silent agent (private_concern ≥ 0) flips to VOICE when neighbour-VOICE ratio > θ_i (Kuran 1995). |
| 7 | `OrgPerformance` | Reward | Per-team knowledge stock `K_k(t) ← (1−δ)K_k(t) + voice_share_k`. |
| 8 | `PsafetyUpdate` | PostStep | ψ_i ← clamp(ψ_i + η·voiced − ν·retaliated). |
| 9 | `ClimateSilence` | PostStep | Aggregate `C(t)` + per-team climate; track the convergence delta. |

The decision mechanism is the **only** mutually-exclusive one — the driver inserts exactly one of `VoiceDecisionRule` (for `rule` / `rule_no_ivt`) or `VoiceDecisionLlm` (for `llm`) per `--llm-mode`.

## Synchronous update

The decision mechanisms snapshot every employee's local context (`ρ_i`, `u_{k(i)}`, `σ`, `ψ_i`) at the start of the Decision phase, compute all new expressions/motives/active-rules from that snapshot, and write them back in one batch. This keeps an LLM call's side effects from leaking into another agent's observation within the same step.

## IVT rules: rule mode vs LLM mode

- **rule / rule_no_ivt** — the VOICE logit `σ(β0 + β_ψ ψ + β_u u + β_σ σ − β_f f − β_ι ι − β_C ρ + β_θ·cascade)` decides VOICE/SILENCE. For silent agents, each of the five rules fires *independently* with probability `σ(k·(ι·w_r − mid))` (a per-rule uniform draw), so the rules fire discriminantly rather than in lockstep. `rule_no_ivt` forces `β_ι = 0` and fires no rules (the IVT layer is ablated).
- **llm** — the five rules are presented in the prompt, each annotated with its salience `ι_i·w_{i,r}`; the LLM returns `{decision, motive, active_rules}` as JSON.

## RNG streams

`derive_seed(root, &[stream])` partitions the randomness:

- stream `0` — world init (employee attributes + network generation),
- stream `1` — engine (scheduler + retaliation Bernoulli + rule-vote Bernoulli),
- stream `2` — retaliation (documentation label; folded into the engine stream),
- stream `3` — the LLM `(agent_id, t)` seed root.

## Two-layer determinism

The socsim core is bit-reproducible; the LLM layer is confined to `VoiceDecisionLlm` and pseudo-determinised by `temperature=0` + a `(agent_id, t)` seed + the `socsim-llm` prompt→response cache.
