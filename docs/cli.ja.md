# CLI リファレンス

[English](cli.md) | [日本語](cli.ja.md)

`detert` バイナリは 4 つのサブコマンドを持つ．すべて `--output-dir`（既定 `results`）配下にタイムスタンプ付き出力ディレクトリを書き，`results/latest` シンボリックリンクを更新する．

## `run`

単一構成の実行．

| フラグ | 既定値 | 意味 |
|--------|--------|------|
| `--llm-mode` | `rule` | `llm` / `rule` / `rule_no_ivt`（排他）． |
| `--n` | — | 総従業員数（`n_teams × team_size` を上書き）． |
| `--n-teams` | `8` | チーム数． |
| `--team-size` | `25` | チームあたり従業員数． |
| `--n-levels` | `3` | 階層レベル数． |
| `--network-model` | `watts-strogatz` | `watts-strogatz` / `erdos-renyi` / `barabasi-albert`． |
| `--network-k` | `6` | Watts–Strogatz k． |
| `--network-beta` | `0.1` | Watts–Strogatz β / Erdős–Rényi p． |
| `--ivt-mean` | `0.55` | IVT 平均強度 ι̅． |
| `--ivt-sd` | `0.20` | IVT 強度の標準偏差． |
| `--beta-psafety` | `1.2` | β_ψ（VOICE ロジット）． |
| `--beta-fear` | `1.5` | β_f． |
| `--beta-ivt` | `2.0` | β_ι（IVT 主効果；較正済み HiCo 点）． |
| `--p-retaliate` | `0.05` | エージェント・ステップ別報復確率． |
| `--shock-t` | `24` | 外生 σ ショックのステップ． |
| `--t-max` | `60` | 最大ステップ． |
| `--runs` | `1` | 独立試行数（出力は最終試行を反映）． |
| `--seed` | `42` | コア層 seed． |
| `--llm-temperature` | `0.0` | LLM 温度． |
| `--llm-seed` | `0` | LLM seed オフセット． |
| `--cache-path` | `.llm_cache/cache.json` | プロンプト→応答キャッシュ（LLM モード）． |

出力：`config.json`，`metrics.csv`，`agents.csv`，`rule_activation.csv`，`llm_meta.json`．

## `sweep`

`β_ι × ψ̄ × seeds` の直積．

| フラグ | 既定値 | 意味 |
|--------|--------|------|
| `--llm-mode` | `rule` | 意思決定モード． |
| `--beta-ivt-min/max/step` | `0.0 / 1.6 / 0.2` | β_ι グリッド． |
| `--psafety-mean-values` | `0.3,0.5,0.7` | ψ̄ 軸の値． |
| `--runs` | `5` | セルあたり試行数． |
| `--t-max` | `60` | 最大ステップ． |
| `--seed` | `42` | ベース seed． |

出力：`sweep_config.json`，`sweep_summary.csv`（セル × 試行ごとに 1 行）．

## `ablation`

seed 範囲にわたり意思決定モードを対比する．

| フラグ | 既定値 | 意味 |
|--------|--------|------|
| `--modes` | `rule,rule_no_ivt` | カンマ区切りのモード． |
| `--seed-start` / `--seed-end` | `0` / `30` | 包含的 seed 範囲． |
| `--t-max` | `60` | 最大ステップ． |

出力：`sweep_config.json`，`ablation_summary.csv`；モード別平均 upward silence rate と rule − rule_no_ivt の IVT 主効果（Δ と Cohen's d）を表示する．

## `reproduce`

設計アンカー（HiCo ≈ .50，silence–voice r ≈ −.55，弁別的同時発火 < .50）に対するモード別定常レポート．Table 4 風レポートと ABM ルール発火行列から再現する CFA 系適合度指標の **両方** を得るには Python ツール `detert-tools reproduce` を使う．
